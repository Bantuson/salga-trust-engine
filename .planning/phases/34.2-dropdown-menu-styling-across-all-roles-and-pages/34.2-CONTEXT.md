# Phase 34.2: Dropdown Menu Styling Across All Roles and Pages - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace all native `<select>` elements with a unified glass-styled custom Select component across both dashboards (frontend-dashboard and frontend-public). Currently 24 files in the ops dashboard use native `<select>` which renders as OS-default white/gray boxes on the dark glassmorphism theme. Two custom dropdown components exist (`shared/Select.tsx` and `GlassSelect.tsx`) with overlapping implementations that need consolidation.

</domain>

<decisions>
## Implementation Decisions

### Component Unification
- Consolidate to ONE unified component at `shared/components/ui/Select.tsx`
- GlassSelect's lighter glass styling becomes the canonical look (not Select's darker purple)
- Merge keyboard navigation + ARIA attributes from current Select.tsx into the unified component
- Support three size presets: `sm` (navbar/compact), `md` (standard forms), `lg` (prominent filters)
- Full keyboard navigation: Arrow keys, Enter, Space to select, Escape to close
- Full ARIA: `aria-haspopup="listbox"`, `aria-expanded`, `role="option"`, `aria-selected`
- Single-select only — no multi-select support in this phase
- Delete `frontend-dashboard/src/components/settings/GlassSelect.tsx` after migration (no re-export alias)

### Dropdown Panel Styling
- Background: `rgba(255, 255, 255, 0.15)` with `backdrop-filter: blur(24px)` (GlassSelect's glass treatment)
- Border: `1px solid rgba(255, 255, 255, 0.25)` with `border-radius: var(--radius-sm)`
- Hover/focus color: rose/pink `rgba(205, 94, 129, 0.18)` for option hover
- Selected/active item: teal text `var(--color-teal)` with teal background `rgba(0, 191, 165, 0.25)`
- Chevron: muted text color `var(--text-muted)`, not gold or teal
- Trigger border when open: turns teal (`var(--color-teal)`) but NO box-shadow glow
- Max height: `240px` with `overflowY: auto`
- Shadow: `0 8px 32px rgba(0, 0, 0, 0.3)`

### Migration Scope
- ALL 24 files in frontend-dashboard that use native `<select>` must be migrated
- Also include frontend-public dashboard (check for native selects and migrate to shared Select)
- Each migration is a tag+prop swap — replace `<select>` with `<Select>` and `<option>` values become the `options` array prop
- GlassSelect.tsx deleted after all its consumers are migrated

### RoleSwitcher Treatment
- Replace native `<select>` with unified Select using `size="sm"`
- Keep "View as:" label alongside the select
- Auto-position (smart flip): detect available space and flip dropdown upward if near bottom edge
- Full role labels everywhere (trigger and dropdown both show "Chief Financial Officer", not "CFO")
- Container: transparent background (remove the current elevated surface pill/chip styling)

### Claude's Discretion
- Exact padding values for sm/md/lg size presets
- Auto-position flip detection threshold
- Transition/animation timing for dropdown open/close
- Scrollbar styling inside the dropdown panel
- Whether to add a subtle open/close animation

</decisions>

<specifics>
## Specific Ideas

- GlassSelect.tsx is the styling reference — its lighter frosted glass look is preferred over the darker purple in shared/Select.tsx
- Rose/pink is for hover state, teal is for selected/active state — clear distinction between "hovering over" and "currently selected"
- RoleSwitcher should feel integrated into the navbar, not like a standalone form element (hence transparent container)

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `shared/components/ui/Select.tsx`: Has full keyboard navigation (ArrowUp/Down, Enter, Space, Escape) and ARIA listbox pattern. Will be the base for the unified component.
- `frontend-dashboard/src/components/settings/GlassSelect.tsx`: Has the target glass styling (rgba(255,255,255,0.15) + blur(24px)). Simpler API (no keyboard nav). Will be deleted after merge.
- `frontend-dashboard/src/components/rbac/RoleSwitcher.tsx`: Currently uses native `<select>`. Has `ROLE_LABELS` record mapping 18 roles to display names. Needs migration to unified Select with sm size.

### Established Patterns
- CSS variables from `@shared/design-tokens.css` used throughout (--font-body, --text-sm, --radius-md, --color-teal, etc.)
- Inline styles (React.CSSProperties objects) — no CSS modules or Tailwind in the dashboard components
- Click-outside handling via `useRef` + `mousedown` event listener pattern (both Select and GlassSelect use this)
- `data-lenis-prevent` attribute on scrollable dropdowns (seen in NotificationBell) to prevent Lenis smooth scroll interference

### Integration Points
- 24 files in frontend-dashboard import or use native `<select>` — each needs `<Select>` import from `@shared/components/ui/Select`
- frontend-public has its own `CustomSelect` component that may need migration
- `@shared` Vite alias already configured to point to `shared/` directory
- Settings page imports GlassSelect — needs update to import from shared/Select

</code_context>
