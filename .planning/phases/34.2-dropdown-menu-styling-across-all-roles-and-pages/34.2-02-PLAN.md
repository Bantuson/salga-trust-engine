<migration_pattern>
## Standard Migration Pattern

Every native `<select>` migration follows this formula:

**Step 1: Add import**
```tsx
import { Select } from '@shared/components/ui/Select';
```

**Step 2: Replace JSX**
```tsx
// BEFORE
<select
  value={stateValue}
  onChange={(e) => handler(e.target.value)}
  style={{ ... }}
>
  <option value="">All</option>
  <option value="open">Open</option>
</select>

// AFTER
<Select
  value={stateValue}
  onChange={(value) => handler(value)}
  options={[
    { value: '', label: 'All' },
    { value: 'open', label: 'Open' },
  ]}
  size="md"
/>
```

**Key deltas:**
- `onChange`: `(e) => fn(e.target.value)` becomes `(value) => fn(value)`
- `<option>` children become `options` array prop
- Native `style` prop on `<select>` is removed (unified component handles styling)
- For typed casts: `e.target.value as SomeType` becomes `value as SomeType`
- For parseInt: `parseInt(e.target.value, 10)` becomes `parseInt(value, 10)`
- For disabled selects: `disabled` prop is the same on both
</migration_pattern>
