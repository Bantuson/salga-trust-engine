# Phase 34.4: Public Dashboard Municipal Analytics Enhancement - Context

**Gathered:** 2026-03-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Enhance the public TransparencyDashboardPage with richer data visualizations, comparative municipality rankings, time period filtering, and cleanup of unused hooks. The page structure (header, municipality selector, stats sections, privacy notice) remains. No new data sources — work with existing hooks and Supabase views.

</domain>

<decisions>
## Implementation Decisions

### Data Visualizations
- Add Recharts as charting library (lightweight, React-native)
- Resolution rate trends: line charts showing monthly trend per municipality (data already computed in useResolutionRates hook but not rendered)
- Category breakdown: donut/pie chart replacing current horizontal bar chart
- SDBIP achievement: radial/gauge chart per municipality replacing current linear progress bar
- Keep traffic light color scheme (green ≥80%, amber ≥50%, red <50%) across all charts

### Comparative Analytics — Ranking Leaderboard
- Replace current basic "All Municipalities" table with a ranking leaderboard
- Sortable/filterable by metric: Response Time, Resolution Rate, SDBIP Achievement, Total Tickets
- Default sort: Resolution Rate
- Quarterly comparison: show position changes (up/down arrows + delta) compared to previous quarter
- Color-coded tiers: top 3 = green highlight, bottom 3 = red highlight
- Senior/Tier 1 roles report quarterly — public dashboard stats update quarterly to match

### Time Period Controls
- Add financial year selector + quarter filter controls
- South African financial year: April to March (Q1=Apr-Jun, Q2=Jul-Sep, Q3=Oct-Dec, Q4=Jan-Mar)
- Default view: current quarter (even if partial data)
- Quarter filter affects service performance stats and leaderboard
- SDBIP achievement stays annual — not filtered by quarter (inherently annual data)
- Available years: current and previous financial years

### Cleanup
- Delete useHeatmapData hook and all heatmap-related code (types, mock data) — not useful
- Delete useSystemSummary hook and all system summary code — not needed
- Keep current page header ("Municipal Transparency Dashboard" + subtitle) as is

### Claude's Discretion
- Recharts chart styling and color palette (align with existing teal/glassmorphism theme)
- Responsive breakpoints for chart layouts
- Loading skeleton design for charts
- Exact quarter selector UI component (tabs vs dropdown)
- How to handle periods with zero data gracefully

</decisions>

<specifics>
## Specific Ideas

- Quarterly reporting cycle aligns with senior municipal role reporting cadence
- Financial year convention is standard SALGA/government — must use April-March, not calendar year
- Position change arrows in leaderboard add engagement and accountability pressure

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `GlassCard` component: used throughout dashboard, elevate variant available
- `CustomSelect` component: could be used for year/quarter selector dropdowns
- `useResolutionRates` hook: already computes monthly trend data (`.trend` array with `{month, rate}`) — just not rendered
- `useSdbipAchievement` hook: fetches from FastAPI `/api/v1/public/sdbip-performance`
- `useResponseTimes` / `useCategoryBreakdown` hooks: query Supabase `public_ticket_stats` view directly
- Color utilities: `getResponseColor()`, `getRateColor()` already exist in ServicePerformanceStats

### Established Patterns
- Supabase direct queries from public frontend (no FastAPI dependency for most stats)
- Mock data fallback on Supabase errors (graceful degradation)
- CSS variables for theming (`--text-primary`, `--text-secondary`, `--color-teal`)
- Inline styles pattern used throughout (not CSS modules)

### Integration Points
- TransparencyDashboardPage.tsx — main orchestration component, needs new sections
- ServicePerformanceStats.tsx — refactor "All Municipalities" view into leaderboard, add charts
- usePublicStats.ts — add quarter/year filtering params to hooks, remove heatmap/summary hooks
- types/public.ts — may need new types for leaderboard ranking data

</code_context>
