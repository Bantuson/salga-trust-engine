## Summary

Phase 34.4 enhances the public `TransparencyDashboardPage` with Recharts visualizations and comparative ranking features. The key insight is that **Recharts 2.15.4 is already installed** in `frontend-public/package.json` and **the project already has working Recharts patterns** in `frontend-dashboard/src/components/dashboard/` (VolumeChart, SLAComplianceChart, TeamWorkloadChart). There is zero new dependency work needed.

The codebase has strong foundations: `useResolutionRates` already computes a `.trend` array (`{month, rate}[]`) that is simply not rendered, the `GlassCard` component is available, CSS variables (`--color-teal`, `--text-primary`, `--chart-grid-stroke`) are defined in shared design tokens, and the `CustomSelect` component (with `data-lenis-prevent`) already handles the glassmorphism dropdown pattern. The main work is adding new chart components, implementing the leaderboard, adding time period controls with SA financial year logic, extending hooks with quarter filtering, and deleting the two unused hooks plus their types and mock data.

Quarter filtering must be computed client-side from the existing `public_ticket_stats` Supabase view — the view already has `report_date` which enables the quarter calculation. The SA financial year April-March convention is a pure date arithmetic problem that belongs in a utility function.

**Primary recommendation:** Build three new chart components using established Recharts patterns from `frontend-dashboard/`, extend existing hooks with quarter/year params, implement leaderboard as a sortable table with ranking delta logic, and delete `useHeatmapData` / `useSystemSummary` with their dependent types and mock data.

### Pattern 5: SA Financial Year Utility
**What:** Pure function to compute quarter and year boundaries. Must use April-March convention.
**When to use:** `TimePeriodControls` component and hook filtering logic.
**Example:**
```typescript
// Source: SA government convention (April 1 – March 31)
function getSAFinancialYear(date: Date): string {
  // If month is Jan-Mar (0-2), financial year started in previous calendar year
  const year = date.getMonth() < 3 ? date.getFullYear() - 1 : date.getFullYear();
  return `${year}/${(year + 1).toString().slice(2)}`;  // "2025/26"
}

function getQuarterBounds(year: string, quarter: 1 | 2 | 3 | 4): { start: string; end: string } {
  const fyStart = parseInt(year.split('/')[0]);  // 2025
  const quarters = {
    1: { start: `${fyStart}-04-01`,   end: `${fyStart}-06-30` },
    2: { start: `${fyStart}-07-01`,   end: `${fyStart}-09-30` },
    3: { start: `${fyStart}-10-01`,   end: `${fyStart}-12-31` },
    4: { start: `${fyStart + 1}-01-01`, end: `${fyStart + 1}-03-31` },
  };
  return quarters[quarter];
}

function getCurrentSAQuarter(date: Date): 1 | 2 | 3 | 4 {
  const month = date.getMonth() + 1;  // 1-12
  if (month >= 4 && month <= 6)  return 1;
  if (month >= 7 && month <= 9)  return 2;
  if (month >= 10 && month <= 12) return 3;
  return 4;  // Jan-Mar
}
```
March 2026 is Q4 of FY 2025/26. Default view = Q4 2025/26.

## Common Pitfalls

### Pitfall 1: RadialBarChart startAngle/endAngle Convention
**What goes wrong:** Using `startAngle={0} endAngle={360}` gives a full circle; the gauge intent needs `startAngle={180} endAngle={0}` (semi-circle opening downward like a speedometer).
**Why it happens:** Recharts uses degrees measured counter-clockwise from 3 o'clock position, which is non-intuitive.
**How to avoid:** Use `startAngle={180} endAngle={0}` for semi-circle gauge. Set `cy="80%"` to shift center upward so the gauge sits well in the card.
**Warning signs:** Gauge appears as a full circle or renders in wrong orientation.

### Pitfall 2: PieChart Legend Overflow in Small Containers
**What goes wrong:** The default Recharts `Legend` renders below the chart and can overflow a fixed-height container.
**Why it happens:** Recharts default legend position is `bottom` and uses absolute positioning.
**How to avoid:** Either omit the built-in `Legend` and render a custom legend div below the chart (matches project inline-style pattern), or use `<Legend layout="vertical" align="right" verticalAlign="middle" />`.
**Warning signs:** Chart content is clipped; legend items stack outside the card boundary.

### Pitfall 3: Quarter Boundary for "Current Quarter" Default
**What goes wrong:** March 2026 is Q4 of FY 2025/26 (Jan 1 – Mar 31). A naive calendar-quarter calculation (Q1=Jan-Mar) returns "Q1 of 2026" which is wrong for SA government reporting.
**Why it happens:** SA financial year starts April 1, not January 1.
**How to avoid:** Use the `getSAFinancialYear` and `getCurrentSAQuarter` utility functions above. Test with `new Date('2026-03-04')` — expected output: FY=2025/26, Q=4.
**Warning signs:** Default quarter shown is Q1 when it should be Q4.

### Pitfall 4: Stale Hook Data When Quarter Changes
**What goes wrong:** Hook data does not refresh when quarter/year selector changes if the dependency array is incomplete.
**Why it happens:** `useEffect` dependency array must include the quarter and year params, not just `municipalityId`.
**How to avoid:** Add `quarter` and `financialYear` parameters to `useResponseTimes` and `useResolutionRates` hooks, include them in the `useEffect` dependency array. Follow the same pattern as the existing `months` param in `useResolutionRates`.

### Pitfall 5: Deleting Types Still Referenced by Leaderboard Mock Data
**What goes wrong:** Removing `HeatmapPoint` and `SystemSummary` from `types/public.ts` without also removing their imports in `mockDashboardData.ts` and `usePublicStats.ts` causes TypeScript errors.
**Why it happens:** Three-location deletion: types file, mock data file, and hooks file.
**How to avoid:** Delete in order — mock data file first, then hooks file, then types file (or delete all in one atomic commit). The grep locations are: `types/public.ts` (definition), `mockDashboardData.ts` (import + `mockHeatmapData` export), `usePublicStats.ts` (import + `useHeatmapData` + `useSystemSummary` exports + `mockHeatmapData` import).

### Pitfall 6: Previous Quarter Data for Leaderboard Delta
**What goes wrong:** The "previous quarter" for rank comparison has no real data source — it must be derived from mock data for now (the real `public_ticket_stats` view only has `report_date`, not a pre-computed quarterly aggregate).
**Why it happens:** Quarterly rank history requires aggregating by quarter range, which is a client-side computation from row-level data.
**How to avoid:** For mock data, pre-populate `previous_rank` values in the mock leaderboard. For real data, compute previous quarter bounds using `getQuarterBounds(year, prevQ)` and aggregate two parallel queries or sequential aggregations. Keep rank delta calculation in a pure function so it is testable.

// Source: Recharts LineChart API + established VolumeChart.tsx project pattern
import {
  LineChart, Line, XAxis, YAxis, CartesianGrid, Tooltip,
  ResponsiveContainer,

import type { MonthlyTrend } from '../types/public';

interface ResolutionTrendChartProps {
  trend: MonthlyTrend[];
  municipalityName: string;
}

export function ResolutionTrendChart({ trend, municipalityName }: ResolutionTrendChartProps) {
  if (trend.length === 0) {
    return (
      <div style={{ textAlign: 'center', padding: '2rem', color: 'var(--text-secondary)', fontSize: '0.875rem' }}>
        No trend data available
      </div>
    );
  }

  return (
    <ResponsiveContainer width="100%" height={200}>
      <LineChart data={trend} margin={{ top: 8, right: 16, left: 0, bottom: 0 }}>
        <CartesianGrid strokeDasharray="3 3" stroke="var(--chart-grid-stroke)" />

          dataKey="month"
          tick={{ fill: 'var(--chart-axis-text)', fontSize: 10 }}
          tickFormatter={(m) => m.slice(5)}  // "2026-02" → "02"
        />
        <YAxis
          tick={{ fill: 'var(--chart-axis-text)', fontSize: 10 }}
          domain={[0, 100]}
          tickFormatter={(v) => `${v}%`}

          contentStyle={{
            backgroundColor: 'rgba(163, 72, 102, 0.95)',
            border: '1px solid var(--glass-border)',

type SortKey = 'resolution_rate' | 'avg_response_hours' | 'sdbip_achievement_pct' | 'total_tickets';

function getRankDeltaIcon(delta: number | null): string {
  if (delta === null) return '';
  if (delta > 0) return '▲';   // improved position (moved to higher rank)
  if (delta < 0) return '▼';   // dropped position
  return '—';
}

function getRankDeltaColor(delta: number | null): string {
  if (delta === null) return 'var(--text-muted)';

  if (delta < 0) return '#ef4444';

}

  return {};

  municipalityId?: string,
  financialYear?: string,  // "2025/26"
  quarter?: 1 | 2 | 3 | 4,

  // ... existing state setup ...

  useEffect(() => {

      }
    }

## Open Questions

1. **Previous quarter rank data source**
   - What we know: Current `public_ticket_stats` rows have `report_date`. We can aggregate two periods.
   - What's unclear: Whether there is enough historical data in the Supabase view for meaningful previous-quarter rankings. In mock data, we can always pre-populate previous ranks.
   - Recommendation: Use mock data for previous ranks with realistic delta values. Compute from real data only when Supabase has multi-quarter history. Gracefully show `null` delta when prior period has no data.

2. **Quarter tabs vs dropdown UI**
   - What we know: `CustomSelect` is available and styled correctly. Tabs would require new button-group component.
   - What's unclear: Which is more scannable at mobile widths.
   - Recommendation: Use `CustomSelect` for both financial year and quarter selectors. CustomSelect already has `data-lenis-prevent`, keyboard nav, and glassmorphism style. No new component needed. The `placeholder` can be "Q1 (Apr–Jun)", "Q2 (Jul–Sep)", etc.
