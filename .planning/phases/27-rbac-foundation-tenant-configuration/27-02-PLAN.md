**1. Create `src/schemas/department.py`:**

Pydantic v2 schemas:

```python
from pydantic import BaseModel, Field
from uuid import UUID
from datetime import datetime

class DepartmentCreate(BaseModel):
    name: str = Field(..., min_length=2, max_length=150)
    code: str = Field(..., min_length=2, max_length=20, pattern="^[A-Z][A-Z0-9_]*$")  # e.g., "FIN", "INFRA"
    parent_department_id: UUID | None = None
    assigned_director_id: UUID | None = None
    display_order: int = 0

class DepartmentUpdate(BaseModel):
    name: str | None = None
    code: str | None = Field(None, min_length=2, max_length=20, pattern="^[A-Z][A-Z0-9_]*$")
    parent_department_id: UUID | None = None
    assigned_director_id: UUID | None = None
    display_order: int | None = None
    is_active: bool | None = None

class DepartmentResponse(BaseModel):
    id: UUID
    tenant_id: str
    name: str
    code: str
    parent_department_id: UUID | None
    assigned_director_id: UUID | None
    assigned_director_name: str | None = None  # Joined from users table
    is_active: bool
    display_order: int
    is_valid: bool  # True if assigned_director_id is not None
    created_at: datetime
    updated_at: datetime | None

    model_config = {"from_attributes": True}

class OrganogramNode(BaseModel):
    """Recursive tree node for organogram visualization."""
    id: UUID
    name: str
    code: str
    director_name: str | None = None
    director_role: str | None = None
    children: list["OrganogramNode"] = []

class TicketCategoryMappingCreate(BaseModel):
    department_id: UUID
    ticket_category: str = Field(..., min_length=1, max_length=50)

class TicketCategoryMappingResponse(BaseModel):
    id: UUID
    department_id: UUID
    department_name: str | None = None
    ticket_category: str
    model_config = {"from_attributes": True}

class MunicipalitySettingsUpdate(BaseModel):
    category: str | None = Field(None, pattern="^[ABC]$")  # A, B, or C
    demarcation_code: str | None = Field(None, max_length=20)
    sdbip_layers: int | None = Field(None, ge=1, le=5)
    scoring_method: str | None = Field(None, pattern="^percentage$")  # Only valid value in v2.0

class MunicipalitySettingsResponse(BaseModel):
    id: UUID
    name: str
    code: str
    province: str
    category: str | None
    demarcation_code: str | None
    sdbip_layers: int
    scoring_method: str
    settings_locked: bool
    financial_year_start_month: int
    model_config = {"from_attributes": True}
```

**2. Create `src/api/v1/departments.py`:**

Router with prefix `/api/v1/departments`, tags=["departments"].

Endpoints:

a) `POST /api/v1/departments/` — Create department. Requires admin role. Validates: parent_department_id belongs to same tenant (if provided), assigned_director_id is a user with section56_director role or higher tier (if provided). Auto-generates code from name if not provided (uppercase first 3-4 chars). Returns DepartmentResponse.

b) `GET /api/v1/departments/` — List all departments for current tenant. Requires any authenticated user. Returns list of DepartmentResponse. Include `assigned_director_name` by joining users table. Compute `is_valid` as `assigned_director_id is not None`.

c) `GET /api/v1/departments/{department_id}` — Get single department. Requires authenticated user.

d) `PUT /api/v1/departments/{department_id}` — Update department. Requires admin. If municipality has PMS active (check via later PMS readiness), prevent deletion of directors (return 409 with message "Cannot unassign director while PMS is active").

e) `DELETE /api/v1/departments/{department_id}` — Soft-delete (set is_active=False). Requires admin. Prevent if PMS is active.

f) `GET /api/v1/departments/organogram` — Return hierarchical tree for current municipality. Query all active departments with their directors. Build tree recursively:
  - Root departments (parent_department_id IS NULL) are top-level nodes
  - Each node includes children departments
  - Include director name and role from users table
  - Return as OrganogramNode structure
  The result is municipality_name at root with departments as children.

g) `POST /api/v1/departments/ticket-category-map` — Create ticket category to department mapping. Requires admin. Body: TicketCategoryMappingCreate. Enforces 1:1 constraint (one category per tenant).

h) `GET /api/v1/departments/ticket-category-map` — List all ticket category mappings. Requires authenticated user.

i) `DELETE /api/v1/departments/ticket-category-map/{mapping_id}` — Remove mapping. Requires admin.

j) `GET /api/v1/municipalities/settings` — Get current municipality's PMS settings. Requires admin. Returns MunicipalitySettingsResponse.

k) `PUT /api/v1/municipalities/settings` — Update municipality PMS settings. Requires admin. If `settings_locked` is True, return 403 with message "Municipality settings are locked. Unlock first to make changes." Body: MunicipalitySettingsUpdate.

l) `POST /api/v1/municipalities/settings/lock` — Lock settings (set settings_locked=True). Requires admin.

m) `POST /api/v1/municipalities/settings/unlock` — Unlock settings (set settings_locked=False). Requires admin. Should require confirmation param in body: `{"confirm": true}`.

For all department queries, use tenant_id from current_user. TenantAwareModel's event listener handles tenant filtering automatically.
