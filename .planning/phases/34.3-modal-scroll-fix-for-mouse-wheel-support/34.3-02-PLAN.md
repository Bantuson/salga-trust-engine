# Plan 34.3-02: Fix remaining dashboard modals (Pattern A) + public frontend dropdowns + verify

## Objective

Complete the `data-lenis-prevent` fix for the remaining 3 dashboard modal files (all Pattern A — separate body div holds `overflowY: auto`) and fix the 2 public frontend files where inline dropdown panels lack the attribute. Then run the full validation suite to confirm phase completion.

**Why:** Plan 34.3-01 covered the 8 teams/ticket modals. Three Pattern A dashboard modals remain (InviteUserModal, CreateDepartmentModal, MunicipalityDetailModal) plus two public frontend dropdown panels (CustomSelect.tsx, CitizenRegisterPage.tsx) that have `overflowY: auto` but no `data-lenis-prevent`.

**Pattern A definition (reminder):** The modal container div has `overflow: hidden` (or no overflow). A separate inner `<div style={styles.body}>` has `overflowY: auto`. The attribute goes on the body div.
