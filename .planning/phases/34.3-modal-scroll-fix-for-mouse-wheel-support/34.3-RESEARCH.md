# Phase 34.3: Modal Scroll Fix for Mouse Wheel Support - Research

**Researched:** 2026-03-03
**Domain:** Lenis smooth scroll + React modal scroll event handling
**Confidence:** HIGH

## Summary

The mouse-wheel-in-modal problem is a well-understood issue with Lenis smooth scroll. Lenis intercepts all `wheel` events on the document to apply smooth scrolling. When a modal is open with an independently scrollable body div (e.g. `overflowY: auto`), the user's mouse wheel events are captured by Lenis and applied to the page scroll instead of the modal content — so the modal body does not scroll via the mouse wheel even though it scrolls via touchpad swipe, scroll bar drag, or keyboard.

Lenis provides a first-class solution: the `data-lenis-prevent` HTML attribute. When placed on a scrollable element, Lenis skips event interception for that element and its subtree. This is the established pattern already used in this codebase for newer modals (CreateIdpModal, CreateSdbipModal, CreatePaModal, DepartmentDetailModal, OrganogramModal, PmsDetailModal) and for the shared Select dropdown panel.

**The bug:** Nine modal components were created before the `data-lenis-prevent` pattern was established (or were created based on the TeamCreateModal reference pattern which itself lacks the attribute). These modals have `overflowY: auto` body divs but no `data-lenis-prevent`, causing mouse wheel scroll to be silently swallowed by Lenis.

**Primary recommendation:** Add `data-lenis-prevent` to the scrollable body element of every modal that is missing it. This is a targeted, low-risk, zero-dependency fix. No library changes, no structural refactoring required.

### Alternatives Considered
| Instead of | Could Use | Tradeoff |
|------------|-----------|----------|
| `data-lenis-prevent` attribute | `data-lenis-prevent-wheel` attribute | `data-lenis-prevent` stops both wheel AND touch; `data-lenis-prevent-wheel` stops only wheel — use the full `data-lenis-prevent` for modal bodies as both wheel and touch should scroll the modal |
| `data-lenis-prevent` attribute | `lenis.stop()` / `lenis.start()` via useLenis | Programmatic approach works but requires React context access in every modal; attribute approach is simpler, self-contained, and requires no prop threading |
| `data-lenis-prevent` attribute | `prevent: (node) => node.id === 'modal'` in LenisProvider options | Works but couples the LenisProvider to specific DOM IDs; brittle; attribute approach is self-documenting and decentralized |

### Pattern 1: Two-Structure Modal (Separate Body Div) — Most Common
**What:** Modal container has `overflow: hidden`, separate inner `<div style={styles.body}>` has `overflowY: auto`. The body div is where `data-lenis-prevent` goes.
**When to use:** All PMS modals, InviteUserModal, CreateDepartmentModal, MunicipalityDetailModal, TeamCreateModal, BulkInviteDialog
**Example (verified from CreateIdpModal.tsx — working reference):**
```tsx
{/* Body */}
<div data-lenis-prevent style={styles.body}>
  {/* content */}
</div>
```
```tsx
// styles.body
body: {
  flex: 1,
  overflowY: 'auto' as const,
  padding: 'var(--glass-card-padding)',
} as React.CSSProperties,
```

### Pattern 2: Scroll on Modal Container — Older Style
**What:** The modal container div itself has `overflowY: auto` (no separate body div). The `data-lenis-prevent` goes on the modal container div that already has `onClick={e.stopPropagation()}`.
**When to use:** TeamDetailModal, MemberDetailModal, AssignmentDetailModal, MemberScheduleModal, ReviewDetailModal, TicketDetailModal
**Example:**
```tsx
<div
  style={styles.modal}
  onClick={(e) => e.stopPropagation()}
  data-lenis-prevent
>
```
```tsx
// styles.modal has:
modal: {
  maxHeight: '85vh',
  overflowY: 'auto' as const,
  // ...
} as React.CSSProperties,
```

### Anti-Patterns to Avoid
- **Putting `data-lenis-prevent` on the overlay div (not the scrollable div):** The overlay is `position: fixed`, `overflow: hidden` — it doesn't scroll. Lenis prevent must be on the element that actually scrolls.
- **Using `data-lenis-prevent` as a JSX prop without a value:** In TSX, `data-lenis-prevent` alone (no value) is the correct form. This is a valid boolean HTML attribute. Do NOT write `data-lenis-prevent={true}`.
- **Adding `data-lenis-prevent` to both overlay and body:** Redundant — only the scrollable element needs it.
- **Using `lenis.stop()` on modal open without `lenis.start()` on close:** If lenis is stopped but never restarted (e.g. modal closes without cleanup), the entire page scroll breaks permanently until refresh.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Prevent lenis from intercepting modal wheel events | Custom wheel event handler with `e.stopPropagation()` | `data-lenis-prevent` attribute | stopPropagation doesn't stop Lenis (it captures at document level); attribute is the intended API |
| Detect when user is scrolling inside modal | Tracking scroll position state | `data-lenis-prevent` attribute | Unnecessary complexity — attribute handles it declaratively |
| Temporarily disable lenis | `lenis.stop()` in useEffect with cleanup | `data-lenis-prevent` attribute | Programmatic approach introduces async/timing risks with cleanup and race conditions |

**Key insight:** Lenis processes events by traversing the element ancestry looking for `data-lenis-prevent`. This means the attribute works passively without any JavaScript event handling overhead.

## Common Pitfalls

### Pitfall 1: Missing the Data Attribute on Correct Element
**What goes wrong:** Developer adds `data-lenis-prevent` to the overlay div (or the modal wrapper with `onClick={e.stopPropagation()}`) instead of the scrollable body div.
**Why it happens:** Confusing which element is "the modal" vs which element scrolls.
**How to avoid:** Find the element with `overflowY: auto` (or `overflow: auto/scroll`) and add `data-lenis-prevent` there.
**Warning signs:** Wheel still doesn't scroll modal content after the fix.

### Pitfall 2: Pattern A vs Pattern B Confusion
**What goes wrong:** Some modals (TeamDetailModal, MemberDetailModal, etc.) scroll the modal container div itself rather than a separate body div. Adding `data-lenis-prevent` to the body div does nothing if there is no body div.
**Why it happens:** Two distinct structural patterns exist in this codebase (older team modals vs newer PMS modals).
**How to avoid:** Check each modal individually. If `styles.modal` has `overflowY: auto`, add `data-lenis-prevent` to the modal div. If `styles.body` has `overflowY: auto`, add it to the body div.
**Warning signs:** Looking at TicketDetailModal which has TWO overflow zones (both `modal` and `tabContent`/inner divs) — may need `data-lenis-prevent` in multiple places.

### Pitfall 3: TeamCreateModal Has a Body Div Without overflow:hidden on Container
**What goes wrong:** TeamCreateModal has both `styles.modal` and `styles.body` but the body div lacks `data-lenis-prevent`. The `styles.modal` does not have `overflowY: auto` (the body div has it at line 376 of TeamCreateModal.tsx).
**Why it happens:** TeamCreateModal is the reference pattern template but was created before this fix.
**How to avoid:** `data-lenis-prevent` goes on the `<div style={styles.body}>` in TeamCreateModal.

### Pitfall 4: Public Frontend VideoModal Has No `data-lenis-prevent`
**What goes wrong:** The `VideoModal` in frontend-public sets `document.body.style.overflow = 'hidden'` but has no `data-lenis-prevent`. This modal shows a video without scrollable content so it is not affected by the bug.
**Why it happens:** Video modal doesn't have a scrollable area so it is not a bug target.
**How to avoid:** Only fix modals that have scrollable content areas with `overflowY: auto`.

### Pitfall 5: Public CustomSelect Dropdown Missing `data-lenis-prevent`
**What goes wrong:** `frontend-public/src/components/CustomSelect.tsx` has a dropdown with `overflowY: auto` but no `data-lenis-prevent` — this is NOT the shared `@shared/components/ui/Select` component.
**Why it happens:** The public frontend has its own custom dropdown (CitizenRegisterPage inline dropdown and CustomSelect.tsx) that pre-dates the shared Select component.
**How to avoid:** The public CustomSelect.tsx dropdown list div needs `data-lenis-prevent` added. The inline dropdown in CitizenRegisterPage.tsx also needs it.
