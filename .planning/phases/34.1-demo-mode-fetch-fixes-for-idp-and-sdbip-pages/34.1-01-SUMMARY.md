## Decisions Made
- DEMO_MODE guard placed before `setLoading(true)` in all three `fetchAll` functions — prevents the one-frame loading flash that occurs when `useState(true)` is the initial state and loading state is set after the guard
- `setLoading(false)` explicitly called in every demo branch before `return` — required because initial state is `true`
- Mutation handler guards placed after parameter validation (`if (!cycleId) return;`, `e.preventDefault();`) rather than as absolute first line — preserves existing short-circuits for invalid state
- Demo data uses Umsobomvu Municipality / Colesburg context and NW water loss KPI — matches the realistic South African municipal theme established in other demo data across the codebase
