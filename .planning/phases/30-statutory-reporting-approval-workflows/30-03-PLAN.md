Add to the same file (after StatutoryReportSnapshot):

```python
class StatutoryDeadline(TenantAwareModel):
    """Statutory reporting deadline for a given financial year.

    Deadlines are computed from the financial_year string, not hardcoded.
    The Celery beat task checks these daily for notification windows.
    """
    __tablename__ = "statutory_deadlines"
    __table_args__ = (
        UniqueConstraint(
            "report_type", "financial_year", "quarter", "tenant_id",
            name="uq_statutory_deadline_type_fy_q"
        ),
    )

    report_type: Mapped[str] = mapped_column(String(20), nullable=False)
    financial_year: Mapped[str] = mapped_column(String(10), nullable=False)
    quarter: Mapped[str | None] = mapped_column(String(2), nullable=True)  # Q1-Q4 for S52; null for others
    deadline_date: Mapped[date] = mapped_column(Date, nullable=False)
    description: Mapped[str] = mapped_column(String(200), nullable=False)

    # Auto-task tracking
    task_created: Mapped[bool] = mapped_column(Boolean, default=False, server_default="false")
    task_created_at: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)
    report_id: Mapped[UUID | None] = mapped_column(ForeignKey("statutory_reports.id"), nullable=True)  # FK to auto-created report

    # Notification flags (prevent duplicate sends)
    notification_30d_sent: Mapped[bool] = mapped_column(Boolean, default=False, server_default="false")
    notification_14d_sent: Mapped[bool] = mapped_column(Boolean, default=False, server_default="false")
    notification_7d_sent: Mapped[bool] = mapped_column(Boolean, default=False, server_default="false")
    notification_3d_sent: Mapped[bool] = mapped_column(Boolean, default=False, server_default="false")
    notification_overdue_sent: Mapped[bool] = mapped_column(Boolean, default=False, server_default="false")
```

**3. Create `src/services/deadline_service.py`:**

`DeadlineService` class with:

- `compute_deadlines(financial_year: str) -> list[dict]`:
  - Parse financial_year "YYYY/YY" to extract start_year (e.g., 2025 from "2025/26")
  - Compute all 7 deadlines dynamically (NO hardcoded date literals):
    ```python
    start_year = int(financial_year.split("/")[0])
    end_year = start_year + 1
    deadlines = [
        {"report_type": "section_52", "quarter": "Q1", "deadline_date": date(start_year, 10, 31),
         "description": f"Section 52 Q1 Performance Report ({start_year})"},
        {"report_type": "section_52", "quarter": "Q2", "deadline_date": date(end_year, 1, 31),
         "description": f"Section 52 Q2 Performance Report ({start_year}/{end_year})"},
        {"report_type": "section_72", "quarter": None, "deadline_date": date(end_year, 1, 25),
         "description": f"Section 72 Mid-Year Assessment ({financial_year})"},
        {"report_type": "section_52", "quarter": "Q3", "deadline_date": date(end_year, 4, 30),
         "description": f"Section 52 Q3 Performance Report ({end_year})"},
        {"report_type": "section_52", "quarter": "Q4", "deadline_date": date(end_year, 7, 31),
         "description": f"Section 52 Q4 Performance Report ({end_year})"},
        {"report_type": "section_46", "quarter": None, "deadline_date": date(end_year, 8, 31),
         "description": f"Section 46 Annual Performance Report ({financial_year})"},
        {"report_type": "section_121", "quarter": None, "deadline_date": date(end_year + 1, 1, 31),
         "description": f"Section 121 Annual Report ({financial_year})"},
    ]
    ```
  - Return the list of deadline dicts

- `async def populate_deadlines(financial_year: str, tenant_id: str, db: AsyncSession) -> list[StatutoryDeadline]`:
  - Call compute_deadlines to get the list
  - For each deadline, INSERT if not exists (check UniqueConstraint; skip duplicates)
  - Set created_by = "system"
  - Return list of created/existing deadline records

- `async def check_and_notify(db: AsyncSession, tenant_id: str) -> dict`:
  - Get today's date
  - Query all StatutoryDeadline records for this tenant where deadline_date >= today - 1 day (include recently overdue)
  - Also query deadlines where notification_overdue_sent=False AND deadline_date < today (catch overdue that weren't flagged)
  - For each deadline, check notification windows:
    - 30 days before: if days_until <= 30 and not notification_30d_sent -> send + set flag
    - 14 days before: if days_until <= 14 and not notification_14d_sent -> send + set flag
    - 7 days before: if days_until <= 7 and not notification_7d_sent -> send + set flag
    - 3 days before: if days_until <= 3 and not notification_3d_sent -> send + set flag
    - Overdue: if days_until < 0 and not notification_overdue_sent -> send + set flag
  - "Send" means BOTH in-app Notification records AND email dispatch:
    - Determine responsible users by role: CFO and Municipal Manager get all deadline notifications
    - Query users with role assignments for cfo, municipal_manager within the tenant
    - **In-app:** Create one Notification record per user per notification event
    - Notification type: DEADLINE_WARNING for 30/14/7/3d, DEADLINE_OVERDUE for overdue
    - Notification title: e.g., "Statutory Deadline: 14 days remaining"
    - Notification message: e.g., "The Section 52 Q1 Performance Report for 2025/26 is due on 31 October 2025. 14 days remaining."
    - Notification link: "/pms?view=statutory-reports" (deep link to reports page)
    - **Email:** For each user with an email address, send an email with the same title/message using `httpx` POST to Supabase Auth admin endpoint or SMTP. Use a helper `_send_deadline_email(user_email: str, subject: str, body: str)` that:
      - Constructs an HTML email body with the notification message
      - Sends via `settings.SMTP_HOST`/`settings.SMTP_PORT` if configured (using `smtplib.SMTP` in a sync context since we're in Celery)
      - Falls back to logging if SMTP not configured (graceful degradation for dev)
      - Never blocks or fails the main notification flow (try/except with logging)
  - Commit all changes
  - Return stats dict: {notifications_sent, deadlines_checked}

- `async def auto_create_report_tasks(db: AsyncSession, tenant_id: str) -> dict`:
  - Query StatutoryDeadline records where task_created=False AND deadline_date is within 30 days from today AND deadline_date > today
  - For each: create a StatutoryReport in DRAFTING status with appropriate fields from the deadline
    - title = deadline.description
    - report_type = deadline.report_type
    - financial_year = deadline.financial_year
    - quarter = deadline.quarter
    - period_start/period_end computed from financial_year + quarter (same logic as StatutoryReportService.create_report)
    - status = DRAFTING
    - created_by = "system_auto_task"
  - Update deadline: task_created=True, task_created_at=now, report_id=new_report.id
  - Skip if a report already exists for that type+FY+quarter (avoid duplicates with existing manual reports)
  - Commit and return stats dict: {tasks_created, deadlines_processed}

**1. Create `src/tasks/statutory_deadline_task.py`:**

Follow the pa_notify_task.py pattern exactly:

```python
"""Celery task for checking statutory deadlines and sending notifications.

Runs daily at 07:00 SAST via Celery Beat. For each tenant:
1. Auto-populates deadline records for the current financial year (if not already populated)
2. Checks each deadline for notification windows (30/14/7/3 days, overdue)
3. Creates Notification records for responsible users (CFO, MM)
4. Auto-creates report drafting tasks 30 days before deadlines (REPORT-09)

Pattern follows src/tasks/pa_notify_task.py:
- asyncio.run() wraps async logic
- Windows event loop compatibility
- Tenant discovery via raw text() SQL
- set_tenant_context() / clear_tenant_context() per tenant
"""
```

- Task name: `"src.tasks.statutory_deadline_task.check_statutory_deadlines"`
- `@app.task(bind=True, name=..., max_retries=3)`
- Determine current financial year from today's date:
  - If month >= 7 (July-December): FY = f"{year}/{str(year+1)[-2:]}" (e.g., "2025/26")
  - If month < 7 (January-June): FY = f"{year-1}/{str(year)[-2:]}" (e.g., "2025/26" for Jan 2026)
- Tenant discovery via raw SQL: `SELECT DISTINCT tenant_id FROM statutory_deadlines` UNION `SELECT DISTINCT tenant_id FROM municipalities WHERE is_active = true` (to populate deadlines for tenants that don't have them yet)
  - Actually, since Municipality is NonTenantModel and doesn't have tenant_id, use: `SELECT DISTINCT tenant_id FROM users WHERE is_active = true` as tenant discovery (same approach works — every active tenant has at least one user)
  - OR simpler: `SELECT DISTINCT tenant_id FROM sdbip_scorecards WHERE is_deleted = false` — tenants with SDBIP data are the ones that need deadline tracking
- Per tenant:
  - `DeadlineService.populate_deadlines(current_fy, tenant_id, db)` — idempotent
  - `DeadlineService.check_and_notify(db, tenant_id)` — creates notifications
  - `DeadlineService.auto_create_report_tasks(db, tenant_id)` — auto-create reports
- Log results per tenant and overall summary
- Retry on failure with exponential backoff
