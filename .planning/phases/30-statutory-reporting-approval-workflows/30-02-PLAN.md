**1. Add completeness validation to `src/services/statutory_report_service.py`:**

Add method `validate_report_completeness(report: StatutoryReport, db: AsyncSession) -> dict`:
- Returns dict with `is_complete` (bool) and `missing_items` (list of strings)
- Checks vary by report type:

For SECTION_52:
- At least 1 SDBIP scorecard exists for the financial_year (missing: "No SDBIP scorecard for {FY}")
- At least 1 SDBIP KPI exists for the financial_year (missing: "No KPIs found for {FY}")
- At least 1 actual submitted for the specified quarter (missing: "No actuals submitted for {quarter}")

For SECTION_72:
- Same as S52 but checks Q1 AND Q2 actuals
- Missing Q1 or Q2 actuals listed separately

For SECTION_46:
- SDBIP scorecard exists
- KPIs exist
- All 4 quarters should have actuals (list which quarters are missing)
- Warn (but don't block) if PA assessments incomplete

For SECTION_121:
- All S46 checks
- Warn if no IDP cycle exists for the period

**Call this from the generate endpoint:** Before dispatching the Celery task, call `validate_report_completeness`. If `is_complete` is False, return 422 with `{"detail": "Report data incomplete", "missing_items": [...]}`. This prevents wasting Celery resources generating empty reports.
