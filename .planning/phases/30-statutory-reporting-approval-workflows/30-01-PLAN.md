**1. Create `src/services/statutory_report_service.py`:**

`StatutoryReportService` class with these async methods:

- `create_report(data: StatutoryReportCreate, user: User, db: AsyncSession) -> StatutoryReport`:
  - Compute `period_start` and `period_end` from financial_year and quarter (or report_type for S72/S46/S121):
    - Parse financial_year "YYYY/YY" to extract start_year (e.g., 2025 from "2025/26")
    - S52 Q1: Jul 1 to Sep 30, Q2: Oct 1 to Dec 31, Q3: Jan 1 to Mar 31 (next year), Q4: Apr 1 to Jun 30 (next year)
    - S72: Jul 1 to Dec 31 (H1)
    - S46: Jul 1 to Jun 30 (full year)
    - S121: Jul 1 to Jun 30 (full year)
  - Validate uniqueness (report_type + financial_year + quarter + tenant_id) — catch IntegrityError -> 409
  - Require quarter for SECTION_52 (422 if missing)
  - Status starts as DRAFTING
  - Set created_by/updated_by from user.id
  - Create AuditLog entry (OperationType.CREATE)
  - Return the created report

- `transition_report(report_id: UUID, event: str, user: User, db: AsyncSession) -> StatutoryReport`:
  - Fetch report by ID (404 if not found)
  - Role gates (enforce via user role check):
    - `submit_for_review`: pms_officer, department_manager, cfo, admin, salga_admin
    - `approve`: municipal_manager, cfo, admin, salga_admin
    - `submit_external`: municipal_manager, admin, salga_admin
    - `table`: municipal_manager, speaker, admin, salga_admin
  - Bind ReportWorkflow state machine: `ReportWorkflow(model=report, state_field="status", start_value=report.status)`
  - `machine.send(event)` — catch `TransitionNotAllowed` -> 409
  - **CRITICAL (REPORT-06):** If event is "approve" (transition to mm_approved):
    - Call `_snapshot_report_data(report, db)` to snapshot all SDBIP data
    - Set `report.approved_by = str(user.id)`, `report.approved_at = datetime.now(timezone.utc)`
  - Create AuditLog entry (OperationType.UPDATE)
  - Commit and return report

- `_snapshot_report_data(report: StatutoryReport, db: AsyncSession) -> StatutoryReportSnapshot`:
  - Assemble all SDBIP KPIs + quarterly targets + actuals for the report's financial_year and quarter scope
  - For S52: filter actuals by specific quarter; for S72: Q1+Q2; for S46/S121: all 4 quarters
  - Build JSON structure: list of dicts with kpi_number, description, unit, baseline, annual_target, quarterly_target, actual_value, achievement_pct, traffic_light_status, variance, department_name
  - Also include PA annual_score summaries if report_type is S46 or S121
  - Serialize to JSON string and store in StatutoryReportSnapshot
  - snapshot_reason = "mm_approved"

- `get_report(report_id: UUID, db: AsyncSession) -> StatutoryReport`: Simple SELECT by ID, 404 if not found
- `list_reports(financial_year: str | None, report_type: str | None, db: AsyncSession) -> list[StatutoryReport]`: Optional filters, order by created_at desc
- `get_report_snapshot(report_id: UUID, db: AsyncSession) -> StatutoryReportSnapshot | None`: Get latest snapshot for a report

- `assemble_report_data(report: StatutoryReport, db: AsyncSession) -> dict`:
  - If report has a snapshot (status >= mm_approved), render from snapshot_data JSON
  - Otherwise, query live SDBIP data (same logic as _snapshot_report_data but returns dict, does not persist)
  - **Municipality branding (REPORT-08):** Query Municipality model by tenant_id to get `municipality_name` and `logo_url`. Include both in the returned dict.
  - Returns dict with keys: municipality_name, logo_url (str or None), report_title, financial_year, quarter, period_start, period_end, kpis (list), summary_stats (total KPIs, green/amber/red counts, overall achievement %), departments (list of dept summaries)
  - This dict is passed to Jinja2 template rendering

- `POST /{report_id}/transitions` — Transition report status. Body: ReportTransitionRequest. Returns StatutoryReportResponse.
  - Transition to mm_approved requires `require_min_tier(1)` (MM/CFO are Tier 1/2 — use role check inside service, not as dependency).
- `POST /{report_id}/generate` — Trigger PDF/DOCX generation. Dispatches Celery task `generate_statutory_report.delay(str(report_id), str(current_user.id))`. Returns 202 Accepted with `{"task_id": task_id, "message": "Report generation queued"}`.
- `GET /{report_id}/snapshot` — Get snapshot data (for debugging/viewing). Returns ReportSnapshotResponse or 404.
- `GET /{report_id}/download/{format}` — Download PDF or DOCX. format is "pdf" or "docx". Reads from pdf_storage_path / docx_storage_path. Returns FileResponse or 404 if not yet generated.
