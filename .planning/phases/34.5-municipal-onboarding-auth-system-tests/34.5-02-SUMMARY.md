## Decisions Made
- **departments step gate:** The departments step uses `disabled={departments.length === 0}` on the Next button (not a validation error element). Test adapted to assert `toBeDisabled()` rather than looking for error text. This matches the actual UI behavior precisely.
- **welcome step button text:** The welcome step renders "Confirm & Continue" (not "Start Setup"/"Next"). Used a flexible regex `/Confirm.*Continue|Start Setup|Get Started|Begin|^Next$/i` to handle all variants robustly.
- **base() vs base.test():** The `base` import from auth fixtures is the test function itself. Correct usage is `base('test name', async ({ page }) => {...})` not `base.test(...)`.
- **reject assertion:** Used a compound assertion (Approve & Onboard count decreased OR rejected badge visible) to cover both UI patterns — the page may either remove the row or update its status badge, both are valid responses to reject.
