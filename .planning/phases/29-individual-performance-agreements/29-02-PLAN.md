## Goal

Evaluators can score individual KPIs per quarter in the quarterly review workflow. The system compiles the annual assessment score from all quarterly scores weighted by KPI weights. Celery notifies evaluators at quarter-start. Completed PA records carry a POPIA retention flag. The frontend shows a Performance Agreements view in the PMS Hub.

## Must-Haves (Goal-Backward)

1. `PAService.add_quarterly_score(pa_kpi_id, data, user, db)` creates `PAQuarterlyScore` with score (0-100), scored_by, scored_at (PA-03)
2. `PAService.compile_annual_score(agreement_id, db)` computes weighted average: `sum(avg_quarterly_score_per_kpi * kpi_weight) / sum(kpi_weights)` and stores in `agreement.annual_score` (PA-04)
3. `POST /api/v1/pa/kpis/{pa_kpi_id}/scores` endpoint for quarterly score submission gated by `require_min_tier(1)` + pms_officer (PA-03)
4. `POST /api/v1/pa/agreements/{id}/compile-score` endpoint triggers annual score compilation (PA-04)
5. `assess` transition in `transition_agreement` guards: at least one PAQuarterlyScore must exist; calls `compile_annual_score()` before setting status (PA-04, PA-05)
6. Celery task `pa_notify_task.notify_pa_evaluators` in beat schedule runs at Q-start (1st of Jan/Apr/Jul/Oct at 08:00 SAST) (PA-03 support)
7. Celery task iterates tenants via `text()` raw SQL (bypasses ORM tenant filter), sets tenant context per-tenant with try/finally (STATE.md pattern)
8. `PerformanceAgreementsPage.tsx` frontend page displays PA list with status badges, KPI table, and score entry form
9. `PmsHubPage.tsx` extended: `PmsView` type includes `'performance-agreements'`, `VIEW_OPTIONS` includes `{ value: 'performance-agreements', label: 'Performance Agreements', createLabel: '+ Create Agreement' }`
10. `GET /api/v1/pa/agreements/{id}/kpis` includes quarterly_scores via `selectinload(PAKpi.quarterly_scores)` for frontend display
11. Duplicate quarterly score (same pa_kpi_id + quarter) returns 409 via IntegrityError catch
12. All new unit tests pass: `pytest tests/test_pms_pa.py -x`

<task id="29-02-1" title="Quarterly Score Submission and Annual Score Compilation in Service + API" status="pending">

### Description

Extend `PAService` with quarterly score submission and annual score compilation methods. Extend the PA API router with score submission and score compilation endpoints. Add `assess` transition guard that requires at least one score.

### Requirements Addressed
- PA-03: Evaluator scores individual KPIs per quarter
- PA-04: System compiles annual assessment score from quarterly scores and KPI weights
- PA-05: assess transition includes score guard and auto-compilation

### Acceptance Criteria

**Service layer additions to `src/services/pa_service.py`:**

- `PAService.add_quarterly_score(pa_kpi_id, data, user, db)`:
  - Validates PAKpi exists (404)
  - Creates `PAQuarterlyScore` with `score=data.score`, `scored_by=str(user.id)`, `scored_at=datetime.now(timezone.utc)`, `notes=data.notes`, `quarter=data.quarter`, `tenant_id` from PAKpi
  - Catches IntegrityError on UniqueConstraint -> 409 (duplicate score for same quarter)
  - Returns created score

- `PAService.compile_annual_score(agreement_id, db)`:
  - Fetches all PAKpis for agreement with `selectinload(PAKpi.quarterly_scores)`
  - For each PAKpi with scores: avg_score = sum(scores) / len(scores)
  - Weighted total: `sum(avg_score * kpi.weight)` / `sum(kpi.weight)`
  - All arithmetic uses `Decimal` (not float) — same as `compute_achievement()` in sdbip.py
  - Division by zero (no KPIs or all zero weight): returns `Decimal("0")`
  - Stores result in `agreement.annual_score`, commits, returns score value

- `PAService.get_kpis_with_scores(agreement_id, db)`:
  - Returns PAKpi list with selectinload of quarterly_scores for frontend display

- **Modify `PAService.transition_agreement`** (from 29-01):
  - Before allowing `assess` event: check at least one PAQuarterlyScore exists for this agreement. If none -> HTTPException 422 "Cannot assess: no quarterly scores submitted"
  - On `assess`: call `compile_annual_score()` before setting `popia_retention_flag=True`

**API additions to `src/api/v1/pa.py`:**

- `POST /kpis/{pa_kpi_id}/scores` -> 201
  - dependencies: `[Depends(require_pms_ready()), Depends(require_min_tier(1))]`
  - Tier 1 because evaluators are MM/ExecMayor; also allow pms_officer via additional `require_role` check in service
  - Body: PAScoreCreate
  - Returns: PAScoreResponse

- `POST /agreements/{agreement_id}/compile-score` -> 200
  - dependencies: `[Depends(require_pms_ready()), Depends(require_min_tier(2))]`
  - Returns: PAResponse (with updated annual_score)

- `GET /agreements/{agreement_id}/kpis` (modify existing from 29-01):
  - Use `get_kpis_with_scores()` instead of `list_kpis()` so response includes quarterly scores
  - Response stays `list[PAKpiResponse]` but PAKpiResponse needs nested `scores: list[PAScoreResponse]` field

**Schema addition to `src/schemas/pa.py`:**

- Add `scores: list[PAScoreResponse] = []` to `PAKpiResponse` for nested quarterly score data

### Implementation Details

Follow `src/services/sdbip_service.py` `submit_actual()` for the score submission pattern. Follow the `compute_achievement()` pattern for Decimal arithmetic in `compile_annual_score()`.

**CRITICAL from STATE.md:**
- Use `selectinload(PAKpi.quarterly_scores)` for the compilation query (not lazy load which fails in async)
- Capture `agreement.id` before commit (MissingGreenlet avoidance)
- IntegrityError catch for duplicate score -> rollback + 409
