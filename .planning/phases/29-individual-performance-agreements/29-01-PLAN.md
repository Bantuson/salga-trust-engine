## Goal

PMS officers can create performance agreements for Section 57 managers linked to a financial year. Each agreement contains PA KPIs linked to organizational SDBIP KPIs with individual targets and weights. The agreement follows a 4-state approval workflow (draft -> signed -> under_review -> assessed), with role-gated signing: Municipal Manager signs director PAs, Executive Mayor signs the Municipal Manager's PA.

## Must-Haves (Goal-Backward)

1. `PerformanceAgreement` model exists inheriting `TenantAwareModel` with `financial_year`, `section57_manager_id` (FK users.id), `manager_role`, `status`, `annual_score`, `popia_retention_flag`, `popia_departure_date` fields and `UniqueConstraint("section57_manager_id", "financial_year", "tenant_id")`
2. `PAKpi` model exists with `agreement_id` FK, `sdbip_kpi_id` FK, `individual_target`, `weight`, `description` and `UniqueConstraint("agreement_id", "sdbip_kpi_id")`
3. `PAQuarterlyScore` model exists with `pa_kpi_id` FK, `quarter`, `score`, `scored_by`, `scored_at`, `notes` and `UniqueConstraint("pa_kpi_id", "quarter")`
4. `PAWorkflow` state machine (python-statemachine 3.0.0) with states draft/signed/under_review/assessed and transitions sign/open_review/assess
5. Pydantic schemas: `PACreate`, `PAKpiCreate`, `PAScoreCreate`, `PATransitionRequest`, `PAResponse`, `PAKpiResponse`, `PAScoreResponse` with `financial_year` YYYY/YY validation
6. `PAService` class with `create_agreement`, `get_agreement`, `list_agreements`, `add_kpi`, `list_kpis`, `transition_agreement` methods
7. `POST /api/v1/pa/agreements` creates PA in draft status (PA-01)
8. `POST /api/v1/pa/agreements/{id}/kpis` adds KPI with sdbip_kpi_id validation and weight sum check (PA-02)
9. `POST /api/v1/pa/agreements/{id}/transitions` applies state machine transitions with role-gated signing (PA-05, PA-06)
10. Signing role gate: MM signs director PA (`manager_role=section57_director`), Executive Mayor signs MM PA (`manager_role=municipal_manager`); admin/salga_admin bypass (PA-06)
11. Duplicate PA (same manager + FY + tenant) returns 409 Conflict
12. Weight sum > 100 returns 422 on KPI creation
13. Alembic migration creates `performance_agreements`, `pa_kpis`, `pa_quarterly_scores` tables with indexes and constraints
14. Models registered in `src/models/__init__.py`; router registered in `src/main.py`
15. Unit tests pass: `pytest tests/test_pms_pa.py -x`

<task id="29-01-3" title="PA Service Layer" status="pending">

### Description

Create `src/services/pa_service.py` with `PAService` class implementing CRUD, KPI management, and state machine transitions with role-gated signing.

### Requirements Addressed
- PA-01: create_agreement creates PA in draft, enforces unique constraint
- PA-02: add_kpi validates sdbip_kpi_id FK, enforces weight sum <= 100
- PA-05: transition_agreement uses PAWorkflow with start_value= binding
- PA-06: Signing role gate based on manager_role field

### Acceptance Criteria
- `PAService.create_agreement(data, user, db)`:
  - Creates `PerformanceAgreement` with `status=PAStatus.DRAFT`, `tenant_id=user.tenant_id`, `created_by=str(user.id)`
  - Validates `section57_manager_id` exists in users table (422 if not found)
  - Catches `IntegrityError` on unique constraint -> HTTPException 409 Conflict
  - Returns created agreement
- `PAService.get_agreement(agreement_id, db)`: returns PA or None (404 in router)
- `PAService.list_agreements(financial_year, db)`: list PAs, optional financial_year filter
- `PAService.add_kpi(agreement_id, data, user, db)`:
  - Validates agreement exists (404)
  - Validates sdbip_kpi_id exists in sdbip_kpis table (422)
  - Computes current weight sum for agreement's KPIs; if adding this KPI would exceed 100 -> HTTPException 422
  - Creates `PAKpi` with `tenant_id=agreement.tenant_id`, `created_by=str(user.id)`
  - Returns created KPI
- `PAService.list_kpis(agreement_id, db)`: list PA KPIs for agreement
- `PAService.transition_agreement(agreement_id, event, user, db)`:
  - Loads agreement (404 if not found)
  - **Sign event role gate**: If `event == "sign"`:
    - If `agreement.manager_role == ManagerRole.SECTION57_DIRECTOR`: allowed = {MUNICIPAL_MANAGER, ADMIN, SALGA_ADMIN}
    - If `agreement.manager_role == ManagerRole.MUNICIPAL_MANAGER`: allowed = {EXECUTIVE_MAYOR, ADMIN, SALGA_ADMIN}
    - If user.role not in allowed: HTTPException 403
  - Creates PAWorkflow with `start_value=agreement.status` (CRITICAL: never omit start_value)
  - `machine.send(event)`, catches `TransitionNotAllowed` -> HTTPException 409
  - If `event == "assess"`: sets `agreement.popia_retention_flag = True`
  - Captures `agreement_id` local variable before commit (avoid MissingGreenlet)
  - Returns updated agreement

### Implementation Details

Follow `src/services/sdbip_service.py` pattern exactly. Import `TransitionNotAllowed` from `statemachine.exceptions`. Import `PAWorkflow`, `ManagerRole` from `src.models.pa`. Import `SDBIPKpi` from `src.models.sdbip` for FK validation. Use `select()` queries for FK validation (not just FK constraint) for SQLite compatibility.

**CRITICAL from STATE.md:**
- Always use `start_value=agreement.status` when instantiating PAWorkflow
- Capture `agreement_id = agreement.id` before `db.commit()` to avoid MissingGreenlet
- Weight sum enforcement at service layer (NOT DB constraint) for SQLite test compatibility
- FK validation via SELECT then 422 (not DB FK violation which gives opaque 500 in SQLite)
